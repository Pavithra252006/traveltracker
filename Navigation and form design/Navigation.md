# Navigation Design

## Project Title

Travel Tracker: A Mobile App for Capturing Trip Information

## Navigation Flow

The application navigation flow describes how users move between different screens of the Travel Tracker mobile application.

```text
Login Page
     ↓
Dashboard
     ↓
Start Trip
     ↓
Trip Tracking
     ↓
Trip Summary
     ↓
Travel History
```

## Detailed Navigation Structure

```text
Login
  ↓
Dashboard
 ├── Start Trip
 │      ↓
 │   GPS Tracking
 │      ↓
 │   End Trip
 │      ↓
 │   Trip Summary
 │
 ├── Travel History
 │
 ├── Profile
 │
 └── Logout
```

## Screen Descriptions

### Login Screen

Allows users to enter their credentials and access the application.

### Dashboard Screen

Displays the main menu and provides access to all application features.

### Start Trip Screen

Allows users to begin a new trip and enable GPS tracking.

### Trip Tracking Screen

Records travel information such as location, distance, and duration.

### Trip Summary Screen

Displays trip details after the trip is completed.

### Travel History Screen

Shows previously recorded trips and travel statistics.

### Profile Screen

Allows users to view and update personal information.

### Logout

Ends the current user session and returns to the Login screen.

## Expected Outcome

The navigation design ensures smooth movement between application screens and provides an intuitive user experience for managing and tracking trips.
